# 변수

*하나의 값을 기억하기 위해서 확보한 메모리 공간, 또는 그 메모리 공간을 기억하는 이름*



- 컴퓨터는 메모리에 저장된 정보를 cpu에서 처리하는 식으로 작업을 하게 된다. 즉 컴퓨터가 계산을 하기 위해서는 먼저 메모리에 계산을 하기위한 정보들이 저장이 되어야 한다는 뜻이다.  

- 저장공간인 메모리의 최소 단위는 1바이트로 셀이라고도 부른다. 각각의 셀은 고유의 주소를 갖는다.  이 메모리 주소는 메모리 공간의 위치를 나타내며 0부터 시작하여 메모리의 크기만큼 정수로 표현된다

- 저장된 데이타를 셀의 주소로 직접 접근하면 오류를 발생시킬수 있고, 주소는 코드가 실행될때마다 메모리 상황에 따라 임의로 지정된다. 때문에 메모리 주소를 통해 데이터에 접근하는 것은 올바른 방법이 아니다.

- 따라서 값의 위치를 가리키는 상징적인 이름을 사용한다. 이것이 변수

- 할당(assignment 대입, 저장) : 변수에 값을 저장하는 것

  참조(reference) 					  : 변수에 저장된 값을 읽어 들이는 것

- 변수의 이름으로 변수의 값을 유추하는데 도움을 줄수있다. 변수 이름은 심사숙고해서



## 식별자

*식별자는 어떤 값을 구별하여 식별해낼 수 있는 고유한 이름*

- 식별자는 메모리주소를 기억해야한다. 즉 식별자는 메모리 주소와 매핑관계를 가지며 메모리는 이 매핑 정보도 저장해야한다.

```
var a = 30;
```

> 변수 a 에 30이란 값을 저장시켰을때 메모리는 30이란 값을 저장하고, a라는 식별자가 30이 저장되어 있는 메모리주소를 가르키고 있다는 정보도 저장해야 한다.

- 메모리 상에 존재하는 어떤 값을 식별할 수 있는 이름은 모두 식별자라고 부른다.
- 네이밍 규칙 준수



## 변수 선언(declaration)

*명령어를 사용하여 자바스크립트 엔진에 변수 식별자의 존재를 알린다*

- var , let , const 키워드를 사용하여 변수를 선언

> keyword는 약속된 동작을 수행하게 하는 명령어

- 2단계를 거쳐 변수를 선언한다

  - 선언 단계(Declaration phase) : 변수 이름을 등록하여 자바스크립트 엔진에 변수의 존재를 알린다.

  - 초기화 단계(Initialization phase) : 값을 저장하기 위한 메모리 공간을 확보하고 암묵적으로 `undefined`를 할당한다.

    > 최기화단계를 거치지 않으면 이전에 다른 애플리케이션이 사용했던 값이 남아 있을수 있다. 변수를 선언하지 않고 변수를 참조하면 쓰레기 값이 나올수 있다.



## 변수 선언의 실행 시점과 변수 호이스팅

- 자바스크립트는 인터프리터 방식으로 코드를 한줄씩 읽고 실행시킨다.

  하지만 변수 선언으면 자바스크립트가 코드를 읽기 이전단계에서 먼저 실행된다(런타임 이전). 런타임 이전단계인 코드의 평가 과정에서 모든 선언문을 찾아내어 먼저 실행 한다. 이후 모든 선언문을 제외한 코드를 순차적으로 실행한다.

- 변수 호이스팅 : 변수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징



## 값의 할당

*할당연산자(=)을 사용하여 우변의 값을 좌변의 변수에 할당*

```
var score;
score = 80;
```

> 변수 선언은 런타임 이전에 실행되지만 값의 할당은 런타임에 실행된다



![](https://poiemaweb.com/assets/fs-images/4-8.png)

> var score = 80 ; 다음과 같이 단축표현해도 변수 선언을 코드읽기 이전단계에서 읽고 인터프리터과정에서 변수 값을 할당한다.
>
> 이때 같은 메모리 공간에 변수 값이 할당되는 것이 아니고 undefined로 선언되어있던 메모리 공간을 지우고 새로운 메모리 공간에 할당값을 저장한다.



## 값의 재할당

*이미 값이 할당되어 이는 변수에 새로운 값을 또다시 할당*

> 변수는 선언과 동시에 undefined로 초기화되기 때문에 엄밀히 말하면 처음 할당값도 재할당이다.

- 재할당을 할수 있기 때문에 변수. 재할당이 불가능하면 상수(constant)

![img](https://poiemaweb.com/assets/fs-images/4-10.png)

> 처음 할당과 마찬가지로 기존 메모리 공간을 지우고 새로운 메모리 공간을 확보하고 값을 저장한다

*garbage collector : 주기적으로 메모리 공간을 검사하여 사용되고있지않은 메모리(어떤식별자도 참조하고 있지 않는 메모리 공간)



## 식별자 네이밍 규칙

- 특수문자를 제외한 문자, '_', '$' 포함할수 있다
- 숫자로 시작할수 없다
- 예악어(프로그래밍 언어에서 사용되고 있거나 사용될 예정인 단어들)는 사용할수 없다
- 변수의 경우 한번에 여러개를 선언할수 있지만 가독성이 나쁘다. 권장x
- 유니코드 문자를 허용하지만 알파벳 이외의 문자는 바람직하지 않다.
- 대소문자를 구별한다.
- 의미를 명확히 표현해야 한다.
- 네이밍 컨벤션!일반적으로 변수나 함수 식별자에는 카멜케이스, 생성자 함수와 클래스 식별자에는 파스칼케이스를 사용한다
